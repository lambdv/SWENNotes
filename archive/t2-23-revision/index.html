      • Activation Function

        Step Function vs Sigmoid Function

        The step function was used as the activation function before neurons when we used perceptron

        If we use the step function instead of sigmoid for the activation function for a neuron, we cannot do a gradient search because the step function gives us a flat error which does not change when the increment our parameters. So we don’t know if we should keep increasing or change directions during the search. There is no gradient since the line is flat and not a curve.

        Instead we should use the sigmoid function where the error does change every time we increment our parameters, letting us calculate the gradient and know which direction to go to reach the optimal solution

        Learning Rate

        Size of steps when going down the gradient when trying to find minimum error during training.

        • like KP in PID controller (low value is too slow and high value will overstep)
        • rate should adjust depending on gradient.
          • if slope is large then step big because we are far from the local minimum,
          • if slope is small then step small because we are close to the local minimum

        Standard Deviation

        The range of randomness the initial values of weights and biases are set to

        • when doing a gradient search in a multilayered network, initial values have to be set to random so that during training they will be different which will lead to the minimum
        • if initial values are the same, then the gradient will vanish and lead to improper training since the weights and biases will be the same and will never reach the minimum

        Hidden Neurons

        Number of neurons in the hidden layer, which output’s are feed as inputs to neurons in the outer layer

        • More hidden neurons means more boundary lines
        • More boundary lines allows for more complicated shapes/classification in nth dimensional vector space.

        Sample Size

        Number of inputs from the training set

        Batch

        Number of steps during training where values of weights and biases are remembered and then averaged for the actual step down the gradient

        • 1 batch = online training
          • stepping down gradient each time new weights and biases are found without thinking
        • batch > 1 = batch training
          • stepping down the gradient based on the average value of weights and biases the next n number of steps downwards
        • mini batch
          • batch takes the average values for weights and biases for the whole data set (one epoch), mini batch takes the average values for weights and biases for a certain number of next steps in the same data set (part of a epoch)

        epoch

        one complete pass of the data through the system/algorithm - i.e. we looked through whole dataset.

        • during training we change weights and bias once for each input in the data set once (one epoch)
        • if we tune the optimal value of weights and bias for each input as we go (error for neuron is close to 0) and then move on to the next one, the overall error will NEVER converge
        • to decrease the overall error for the whole data set, we have to change weights and biases for each input once per epoch
      • Error

        the total accuracy of the network

        error for each neuron are squared because they can be positive or negative

        eg: -0.5 + 0.2 = 0.7

        this leads to an inaccurate/misleading total. so squaring the numbers will keep them positive

        eg: -0.5^2 + 0.2^2 = 0.29

        this gives us a number which more accurately represents how accurate the network is overall

		int mid = (low + high)/2; //middle of list
		int comp = value.compareTo(data.get(mid)); //compare target value to middle value
		
		//if target value is middle value, then we found it
		if (comp == 0) return mid;
		//if target value is below middle, recursive call with high = mid
		if (comp < 0) return indexOf(value, data, low, mid);
		//if target value is above middle, recursive call with low = mid+1
		if (comp > 0) return indexOf(value, data, mid+1, high);

				int comp = value.compareTo(data.get(mid));

				if (comp == 0) return mid; //item found in the middle
				if (comp < 0) high = mid; //divide list from middle to right

      • Partially Ordered Trees

        Adding nodes

        1. add node to bottom most right
        1. push node up the tree by swapping what’s above it until its parent has more priority than it

        cost of this is log2(n) n being number of layers

        Removing nodes

        1. replace root with bottom most right node
        1. push node down by swapping what’s below (pick node with most priority out of the two options) it until its in the correct position
      • Heap

        Adding to Heap

        1. insert node at bottom of array
        1. find its parent
          • (10-1)/2 = 4 = fox
        1. if node is bigger than parent then swap
          • 21 > 4
        1. repeat until node is smaller than parent

        Removing from Heap

        1. replace root with “bottom most right”/last node
        1. swap node with its biggest child with the most priority
          • 2*0+1 = 1 = pig
          • 2*0-2 = 2 = eel
          • eel (26) > pig (21) so swap fox with eel
        1. repeat until it tried to swap with an index that doesn’t exit. this means its in the correct position and we can stop

		private Comparator<E> comp;

		}

		}

		}

		}

				if  (value == null)	return false;
				else  {
						data.add(value);
						pushup(data.size()-1);
						return true;
				}
		}

		public E poll() { //move last element into root
				if  (isEmpty())	return null;
				if  (data.size() == 1)	return data.remove(0);
				else  {
						E ans = data.get(0);
						data.set(0, data.remove(data.size()-1));
						pushdown(0);
						return ans;
				}
		}

		private void swap(List<E> data, int from, int to){
				data.set(child, data.set(parent, data.get(child)));
		}

		private void pushup(int child) {
				if  (child == 0) return;
				int parent = (child-1)/2;
				// compare with value at parent and swap if parent smaller
				if  (comp.compare(data.get(parent), data.get(child)) < 0) {
						swap(data, child, parent);
						pushup(parent); //recurse up the tree...
				}
		}

		private void pushdown(int parent) {
				int largeCh = 2*parent+1;
				int otherCh = largeCh+1;
				     // check if any children
				if  (largeCh >= data.size()) return;
				     // find largest child 
				if  (otherCh < data.size() &&   comp.compare(data.get(largeCh), data.get(otherCh)) < 0 )
				largeCh = otherCh;
				     // compare with largest child, and swap if smaller
				if  (comp.compare(data.get(parent), data.get(largeCh)) < 0) {
						swap(data, largeCh, parent);
						pushdown(largeCh); //recurse down the tree...
				}
		}
