      • Collisions
        • Time splitting

          Fast moving objects

          A fast moving object can go through a wall if its position is updating faster than the time it takes for the code to register if it collided with the wall
          bullet is fast and goes through the window because between t0 and t1 it moved its position past the window in 1 time tick and wasnâ€™t able to register a collision because it was too fast
          • Possible solutions to prevent this:
            • movement volume collision test
            • Constraint on speed of objects
              • (fastest object moves smaller distance than thinnest object)
              • May not be practical for all games
            • Reduce simulation step size
              • Adds overhead since more computation
              • Note, can try different step size for different objects

          Intersection Testing

          giving an object collision padding between its current position and the position it will be in the next time tick can be used to check/predict if it will collide with a wall in the next time tick
          • Extrude geometry in direction of movement
          • test if overlap
          • When there is a collision:
            • Move simulation to time of collision
            • Resolve collision
            • Simulate remaining time step
          • Intersection Testing: Sphere-Sphere Collision Example
            • good for circles, not good for shapes like squares

        • Dealing with Complexity

          Bounding volumes

          the simple geometric shape surround the object to act as a hitbox to simplify objects to test

          Example: approximating a spiky object as a sphere

          Minkowski Sum (âŠ•)

          Take the sum of two convex volumes to create a new volume around the object act as collision padding

          Object has collided if its inside the Minkowski Sum

        • Reduced Collision Tests

          Partitioning

          We can partition space so we only have to test objects in the same cell if they have collided to reduce the amount of collision tests that have to be done when we have a lot of objects moving around
          • If N objects, then sqrt(N) x sqrt(N) cells to get linear complexity
          • But what if objects donâ€™t align nicely?
          • What if all objects in same cell? (same as no cells)

          Plane Sweep

          Instead of checking if an object is coliding with both X and Y axics, if we know an object stays in place, we can record the bounds of objects along axes.
          • in this case we donâ€™t need o check the y axis for object B because of the air gap
