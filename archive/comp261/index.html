        a tree containing grammar/rules as nodes used to parse your formal language

        • nodes with children is an element that is holding other elements (non terminals)

        a Concrete Parse Tree that only shows the important data.

        • a tree representation of an syntactic structure of text.
        Concrete parse tree has too much information, to render the webpage we don’t need all that information. EG: ever HEAD will contain terminals and we only are about the terminal: TITLE, so we can use HEAD to find TITLE
        • only remember the top level rules apart from the bottom level since there is where all the important data is

          representing the tree like this it will be easy to do a depth first search to get the real data easily

        Top Down Recursive Descent Parser

        recursive algorithm that travels the tree and parses the language

        • only works for one form of language its build for
        • looks at next token
        • checks what token is to decide which branch of the rule to follow
        • fails if token is missing or is of a non matching type
        • requires the grammar rules to be highly constrained: (unambiguous, "LL(1)")

          public void traverseGraph(Node start){
		Set<Node> visited = new HashSet<Node>();
		Queue<Node> fringe = new ArrayDeque<Node>(); (or Stack, or PriorityQueue)
		fringe.offer(start);
				while (!fringe.isEmpty()){
						Node node = fringe.poll();
						if (!visited.contains(node)) {
								visited.add(node);
								process(node);
								for (Node neighbour : node.getNeighbours()){
										if (!visited.contains(neighbour)){
										fringe.offer(neighbour);
								}
						}
				}
		}
}

		Map<Node, List<Edge>> backpointers = new HashMap<>();
		Queue<Item> fringe = new PriorityQueue<Item>();
		Set<Node> visited = new HashSet<Node>();

		while(!fringe.isEmpty()){
				Item item = fringe.poll();

				int lengthToNode = item.length();

						visited.add(node);

								if(!visited.contains(neighbour)){

          when we start with a flow graph, it only has forward edges

          to turn it into a Residual Graph, we add backwards edges for each forward edge with an initial capacity of 0

          when we find a path from a to f

          we set the capacity of the forward edges to forward edge capacity - flow from path

          Using residual graph in path finding process

          given a path from a to f in this flow graph

          the max flow of this path is 1 because the bottleneck is the minimum capacity of an edge in the path

          now we can add reverse edges

          start by setting the capacities for forward edges to (flow through the edge in a path) - (its capacity)

          then we can set the capacity for the reverse edges to the new minimum capacity of the path (flow that we have calculated) and flow remains the same

          now with our new graph which has been processed

          we repeat the process on this graph with a new path

          we keep doing this until there is no path left to find because we can not go through the forward edges since they have no capacity left

          the reverse edges should now cancel out the corresponding forward edge

          the properity of canceling out the forward edge when traversing the reverse edge is so that we can undo our decision of traversing the forward edge when using an algortimn to find the optional path

            Random Surfer model

            Models how someone might browse the web without any particular goal in mind.

            The PageRank theory holds that an imaginary surfer who is randomly clicking on links will eventually stop clicking.

            • user starts at a random node and decides to randomly move to another node or stay at this node
            • this is based on probability, this probability is called a damping factor

            Damping Factor

            The probability, at any step, that the person will continue following links is a damping factor 𝑑. The probability that they instead jump to any random page is 1 − 𝑑.

            When calculating PageRank, pages with no outbound links are assumed to link out to all other pages in the collection.

              divided by N because there are N nodes, this specifies the probability of the user going onto this specific node

              the rest of the formula is a movement multiplied/related to the value of d

              the second part is about a person moves to node A through some other edges

              b is sharing its page rank with a completely, because b is only pointing to a

              c will share its only half of its page rank with a because its pointing to both a and b, so half will be given to a and half will be given to b

              about nodes that have no outlinks
