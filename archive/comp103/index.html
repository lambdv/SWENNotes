        • Priority Queue

          Properties

          • elements are ordered according to their natural order or by comparator
          • cannot store null elements
          • doesn’t allow comparable objects (can’t store same elements)
          • poll, remove and peek works uses the element with the highest priority
        • Applications of Queue and Priority Queue
          • operating systems, network applications, multi user systems
          • handling requests, events, jobs that must be done in order
          • simulations
          • representing queues in the real world (traffic, customers, deliveries)
          • managing the events that must happen in the future
          • program to control delivery of orders or manage customers/clients

          Simulations

          modelling a real world dynamic system (each object has a state that determines its behavior) on a computer

          • such as hospital system

          Tick based simulation

          • time is discrete (measured in ticks)
          • loop increments one tick per iteration
          • for each tick, update the state of every entity in the world by one tick
          • may be inefficient if not very much happens most of the time

          Event based simulation

          • A clock keeps track of time
          • keeps a priority queue of all events that are going to happen
          • each iteration of the loop
            • takes first event off the queue
            • updates all objects affected by the event

        • Notation

          Node

          tree is made up of nodes with links

          Links/Edge

          connects a node to a child node

          • a node may have no, one or several children
          • the node is called the parent of its child nodes
          • nodes that have the same parent are called siblings

          Root node

          node that is the base/start of the tree

          • has no parent nodes
          • each node other than root has exactly one parent

          Leaf node

          a node without children

          • also acts as the conclusion

          Internal node

          any node which is not the root (start) or leaf (end) is the internal

          ancestor

          a node is an ancestor of another node if its above it on the path from the root

            problems with this code:

              there are 2 C, so it will remove both of them when we call the code to remove C

            1. trying to remove an item from the same collection a for each loop is iterating through
            1. cannot remove root node (and its subtree) from a tree

            Fix

            public boolean removePerson(Person tree, String name){
		if (tree == null) return false; 

		Person chToRemove = null;

		for (Person ch : tree.getChildren()){

				if (ch.getName().equals(name)){ chToRemove = ch; break; } //add child to remove collection to remove it later in another loop
				else if (removePerson(ch, name)) { return true; }

		}

		if (chToRemove==null){ return false; }

		tree.removeChild(chToRemove);

		return true;
}

        • Traversal Patterns
          • pattern for recursive depth-first traversing
            to traverseDF(node):
		process node
		for each child of node
				traverseDF (child)
          • pattern for iterative traversing General Trees

            to traverseBF(node)
		put node on queue
		while (queue is not empty)
				dequeue node from queue
				process node
				for each child of node
						put child on queue

            to traverseDF(node)
		put node on stack
		while (stack is not empty)
				pop node from stack
				process node
				for each child of node
						push child on stack
          • pattern for recursive depth-first traversing passing depth and other information down the tree
            to traverseDF(node)
		traverseDF(node, 0, info)

to traverseDF(node, depth, info):
		process node at depth with info
		for each child of node
				traverseDF (child, depth+1, addto(info))

        1. find its parent
          • (10-1)/2 = 4
          • 4 = fox
        1. if node is bigger than the parent then swap indexs
          • 21 > 4
        1. keep going
      • Removing from Heap
        1. remove root (item at index 0)
        1. push the “bottom right” aka last node in the array to the top (index 0)
        1. find the node’s biggest children an swap
          • 2*0+1 = 1
          • 2*0_2 - 2
          • pig and eel are fox’s children
            • eel is bigger than pig (2>1) so swap fox and eel
        1. keep going (find fox’s children and swap with the biggest one)
          • when fox tries to swap with an index that doesn’t exist, that means its in the correct position and can stop

        public class HeapQueue <E> extends AbstractQueue <E> { 

		private List<E> data =  new ArrayList<E>(); //use arraylist instead of array so it can resize
		private Comparator<E> comp;   

		public HeapQueue (Comparator <E> c) { //need comparator to compare priority values
				comp = c;
		}
		
		public boolean isEmpty() {
				return data.isEmpty();
		}

		public int size () {
				return data.size();
		}

		public E peek () {
				if  (isEmpty()) return null;
				else return data.get(0);
		}

		public boolean offer(E value) { //add at the end of the array
				if  (value == null)	return false;
				else  {
						data.add(value);
						pushup(data.size()-1);
						return true;
				}
		}

		public E poll() { //move last element into root
				if  (isEmpty())	return null;
				if  (data.size() == 1)	return data.remove(0);
				else  {
						E ans = data.get(0);
						data.set(0, data.remove(data.size()-1));
						pushdown(0);
						return ans;
				}
		}

		private void swap(List<E> data, int from, int to){
				data.set(child, data.set(parent, data.get(child)));
		}

		private void pushup(int child) {
				if  (child == 0) return;
				int parent = (child-1)/2;
				// compare with value at parent and swap if parent smaller
				if  (comp.compare(data.get(parent), data.get(child)) < 0) {
						swap(data, child, parent);
						pushup(parent); //recurse up the tree...
				}
		}

		private void pushdown(int parent) {
				int largeCh = 2*parent+1;
				int otherCh = largeCh+1;
				     // check if any children
				if  (largeCh >= data.size()) return;
				     // find largest child 
				if  (otherCh < data.size() &&   comp.compare(data.get(largeCh), data.get(otherCh)) < 0 )
				largeCh = otherCh;
				     // compare with largest child, and swap if smaller
				if  (comp.compare(data.get(parent), data.get(largeCh)) < 0) {
						swap(data, largeCh, parent);
						pushdown(largeCh); //recurse down the tree...
				}
		}
}

public int indexOf(String value, List<String> data, int low, int high){ //helper

		if (low >= high){ return -1; } //checks if the item exists in the list

		int mid = (low + high)/2; //finds middle of the list

		int comp = value.compareTo(data.get(mid)); //value returned from compareTo method

		if (comp == 0) { return mid; } //item is in the middle
		else if (comp < 0) { return indexOf(value, data, low, mid); } // if its below the middle the recursive call
		else { return indexOf(value, data, mid+1, high);} // if its above the middle the recursive call
}

		while (low < high){
				int mid = (low + high) / 2; //find the middle of the list

				int comp = value.compareTo(data.get(mid));

				if (comp == 0) return mid; //item found in the middle
				if (comp < 0) high = mid; //divide list from middle to right
				else low = mid + 1; //divide list from middle to left
		}

}
